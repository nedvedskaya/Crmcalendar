# Функция восстановления броней из архива

## Описание
Добавлена возможность восстановления случайно выполненных броней из архива с автоматическим удалением связанных финансовых транзакций.

## Основной функционал

### 1. Восстановление брони
- Кнопка "Восстановить" в каждой архивной брони
- Подтверждение действия через диалоговое окно
- Автоматическое удаление связанной транзакции из финансов
- Возврат брони в активное состояние

### 2. Связь с финансами
При восстановлении брони из архива:
- Система ищет связанную транзакцию по параметрам:
  - Сумма (amount)
  - Название услуги (title)
  - Имя клиента (в sub)
  - Тип операции (income)
- Найденная транзакция автоматически удаляется
- Это гарантирует корректность аналитики доходов

### 3. Визуальные улучшения
- Архивные брони теперь имеют белый фон вместо серого
- Серая полоска сверху карточки для визуального отличия
- Убрана opacity для лучшей читаемости
- Кнопка восстановления с оранжевым градиентом (accent color)

## Технические детали

### Новые функции
```javascript
handleRestoreRecord(clientId, recordId)
```
- Находит клиента и бронь
- Удаляет связанную транзакцию из списка
- Убирает флаги `isCompleted` и `isPaid`

### Изменённые компоненты
- `ClientDetails` - добавлен prop `onRestoreRecord`
- Архивный блок броней - добавлена кнопка восстановления
- Импорт иконки `RotateCcw` из lucide-react

## Использование

1. Откройте карточку клиента
2. Прокрутите до раздела "Бронь"
3. Нажмите на "Архив броней" для раскрытия списка
4. Найдите нужную бронь
5. Нажмите кнопку "Восстановить"
6. Подтвердите действие в диалоговом окне

**Важно:** После восстановления связанная транзакция будет удалена из финансов для сохранения корректности аналитики.

## Безопасность
- Диалоговое окно подтверждения предотвращает случайное восстановление
- Пользователь информируется об удалении транзакции
- Все изменения обратимы (бронь можно снова пометить как выполненную)
